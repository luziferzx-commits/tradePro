# Phase 2: Safety & Risk Engine Report

## Overview
Phase 2 successfully implemented the unified **RiskGuard** system. This system acts as an impenetrable gateway ("ห้ามพอร์ตตายก่อน") for all trading signals before they reach the execution layer.

## Key Deliverables

### 1. Unified `RiskGuard` (`risk/guard.py`)
Centralized all risk checks into a single structured evaluation dictionary.

**Output Structure:**
```json
{
    "allowed": true/false,
    "reason": "String detailing the reason for rejection",
    "risk_amount": 0.0,
    "position_size": 0.0,
    "guard_that_failed": "GUARD_CODE"
}
```

### 2. Hard Guards Implemented
1. **Connection Guard (`CONNECTION`)**: Ensures MT5 terminal is connected.
2. **Account Guard (`LIVE_ACCOUNT_RESTRICTION`, `MIN_EQUITY`)**: Validates `ALLOW_LIVE_TRADING` and ensures equity is above the required baseline.
3. **Parameter Guard (`MISSING_SL`, `MISSING_TP`)**: Explicitly enforces Stop Loss and Take Profit.
4. **Market Session Guard (`SESSION_CLOSED`)**: Verifies `symbol_info.session_deals`.
5. **Spread Guard (`MAX_SPREAD`)**: Ensures spread is within `MAX_SPREAD_POINTS`.
6. **Pre-Trade Slippage Guard (`PRICE_DRIFT_EXCEEDED`)**: Prevents execution if the `current_market_price` drifts from `signal_price` by more than `MAX_SLIPPAGE_POINTS`.
7. **Portfolio Limits (`MAX_TRADES_PER_DAY`, `PORTFOLIO_LIMIT`)**: Caps trades per day and prevents overexposure to single/correlated symbols.
8. **Drawdown Protection (`MAX_DAILY_LOSS`, `MAX_DRAWDOWN`, `CONSECUTIVE_LOSSES`)**:
   - `MAX_DAILY_LOSS`: Calculated strictly from today's MT5 closed `history_deals_get`. If unavailable, gracefully falls back to `EQUITY_FALLBACK`.
   - `MAX_DRAWDOWN`: Absolute calculation from Balance vs Equity.
9. **Strict Size Cap (`RISK_EXCEEDS_CAP`)**: Volume generated by Bayesian sizing is mathematically double-checked against the hard cap (1% Balance).

### 3. Execution Integration (`main.py` & `executor.py`)
- `main.py` explicitly processes `RiskGuard.evaluate_trade()`, and skips execution cleanly if `allowed == False`.
- `execution/executor.py` received a final "last-line safety assertion" which re-verifies that `evaluation["allowed"] == True` and that volume never exceeds the authorized `position_size`.

### 4. Robust Testing Suite (`tests/risk/test_risk_guard.py`)
We achieved a completely stable test pipeline for the new guards using 10 comprehensive Pytest unit tests mocking various MT5 scenarios.
- 10/10 tests passed successfully.
- `pytest --collect-only` remains clean with 0 collection errors out of 266 collected tests.

## Conclusion
The system is now hardened to strictly prevent the account from catastrophic failures before any machine learning or AI inference is even considered. The pipeline is safe and ready for further Multi-Market or Strategy modifications.
